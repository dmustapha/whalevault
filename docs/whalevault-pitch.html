WhaleVault

On-chain privacy infrastructure for DeFi: shield, transfer, and swap without revealing amounts or parties.

W H A T   W E   D O

On-chain transactions are public by default. Every DeFi interaction — deposit, transfer, swap — is permanently traceable. Institutions will not participate in protocols where their positions are visible to competitors. Retail users have no MEV protection once a transaction is broadcast. The data is just there, for anyone who wants it.

WhaleVault puts a cryptographic privacy layer on top of DeFi. Deposit assets into a shielded pool and the connection between your deposit and withdrawal is severed at the proof level. Transfers and swaps happen inside the pool without exposing amounts or counterparties on-chain. The only thing anyone can verify is that the proof is valid.

The privacy primitives ship separately as Veil, a standalone SDK with a Python API backed by Rust cryptography. Any team building a privacy-sensitive application can integrate ZK proofs with a single install. The current deployment runs on Solana. The cryptographic stack — Groth16 proofs on BN254, Poseidon hashing, incremental Merkle trees — is standard infrastructure that ports to any ZK-compatible chain.

H O W   I T   W O R K S

ZK Shielding

Deposit assets and receive a cryptographic note. The note encodes your deposit without revealing the amount. A Groth16 zkSNARK proves the deposit is valid without disclosing which deposit you made or how much it is worth.

Private Transfers

Send shielded assets to any address. The transfer happens inside the pool. No amounts, no sender, no recipient visible on-chain — only a valid proof that the note was spent and a new note was issued to the recipient.

Veil SDK

The privacy primitives ship as a standalone Python package backed by Rust via PyO3. Generate commitments, nullifiers, and Merkle proofs with a clean API. Integrate ZK privacy into any application without writing circuit code. Published on PyPI as veil-solana.

Nullifier System

Each deposit note has a one-time nullifier. Spending a note records the nullifier on-chain, preventing double-spends without revealing which note was used. The Merkle tree tracks pool membership cryptographically, not by address.

P R O J E C T

Open Source

The full codebase is public on GitHub — circuits, Veil SDK, backend, and frontend. Every design decision is visible. Developers can fork the SDK, extend the circuits, or integrate the privacy layer into their own protocol.

Demo

A complete walkthrough from shield through private transfer to unshield is on YouTube. The live deployment on Vercel accepts real devnet transactions with no local setup required.

Veil SDK

Published on PyPI as veil-solana. Any Python developer can install the SDK and start generating ZK proofs in minutes. Documentation at veil-sdk.com. The SDK is independent of the WhaleVault application.

Chain Portability

Groth16 proofs, Poseidon hashing, and incremental Merkle trees are standard across ZK ecosystems. The Solana integration is isolated to the application layer. Porting to EVM, StarkNet, or any ZK-compatible chain means replacing the chain interface, not the cryptographic core.
